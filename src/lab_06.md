# Лабораторна робота №6: Підключення до зовнішнього API

## Фрагменти коду

### Кастомний хук `useFetch.js`
```javascript
import { useState, useEffect } from "react";

export const useFetch = (url) => {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const abortController = new AbortController();

    const fetchData = async () => {
      setIsLoading(true);
      try {
        const response = await fetch(url, { signal: abortController.signal });
        if (!response.ok) {
          throw new Error(`Помилка HTTP: ${response.status}`);
        }
        const result = await response.json();
        setData(result);
        setError(null);
      } catch (err) {
        if (err.name === "AbortError") {
          console.log("Запит скасовано");
        } else {
          setError(err.message);
        }
      } finally {
        setIsLoading(false);
      }
    };

    fetchData();

    return () => {
      abortController.abort();
    };
  }, [url]);

  return { data, isLoading, error };
};
```

### Реалізація тріади станів у `Feed.jsx`
```javascript
// ... import useFetch ...

const Feed = () => {
    const { data: posts, isLoading, error } = useFetch("https://jsonplaceholder.typicode.com/posts");

    // 1. Стан завантаження
    if (isLoading) {
        return <div><h2>Завантаження новин...</h2></div>;
    }

    // 2. Стан помилки
    if (error) {
        return <div><h2 style={{ color: 'red' }}>Сталася помилка: {error}</h2></div>;
    }

    // 3. Стан успішного завантаження (Data)
    return (
        <div>
            <h2>Стрічка новин (API JSONPlaceholder)</h2>
            <div className="posts-grid">
                {posts && posts.slice(0, 15).map(post => (
                    <div key={post.id}>
                        <h3>{post.title}</h3>
                        <p>{post.body}</p>
                    </div>
                ))}
            </div>
        </div>
    );
};
```

## Відповіді на контрольні запитання

1. **Поясніть призначення об’єкта `AbortController` у нашому хуку `useFetch`. Яку загрозу для React-додатку він усуває?**
   `AbortController` використовується для примусового скасування (переривання) незаконченого мережевого запиту `fetch()`. Це усуває загрозу витоку пам'яті (memory leak) та помилки оновлення стану демонтованого компонента. Якщо компонент починає завантаження даних, але користувач швидко переходить на іншу сторінку до завершення запиту, компонент знищується. Без скасування запиту, `fetch` все одно завершиться і спробує викликати `setData()`, що викличе попередження в React про оновлення неіснуючого компонента.

2. **Що таке патерн "Тріада станів" (loading, error, data) при роботі з мережею і чому він є обов’язковим для якісного UX?**
   Мережевий запит ніколи не виконується миттєво і завжди містить ризик невдачі. Тріада станів гарантує надійний зворотний зв'язок з користувачем: 1) `loading` (лоадер/спінер) показує, що додаток "думає" і не завис; 2) `error` повідомляє про проблеми (відсутність інтернету/помилка 404), замість того, щоб показувати пустий білий екран; 3) `data` показує безпосередній контент лише тоді, коли він повністю і безпечно завантажений.

3. **Чому функція виконання мережевого запиту розміщується всередині хука `useEffect`, а не прямо в тілі функціонального компонента?**
   Тіло функціонального компонента відповідає виключно за рендеринг UI і виконується при кожній зміні props чи state. Мережевий запит є зовнішнім побічним ефектом (side-effect). Якщо викликати `fetch` прямо в тілі, то при кожному рендері він буде виконуватися знову, знову спрацьовуватиме встановлення `setPosts()`, що призведе до нового рендера і так до нескінченності. Хук `useEffect` дозволяє контролювати, коли саме виконується запит (наприклад, лише 1 раз при монтуванні, вказавши `[]` в масиві залежностей).

4. **У чому полягають головні переваги використання бібліотеки `axios` у порівнянні з нативним `fetch`?**
   - **Автоматична конвертація в JSON:** axios повертає одразу JavaScript об'єкт (не треба робити `await res.json()`).
   - **Справжня обробка помилок:** axios переходить у блок `catch`, якщо статус сервера поза діапазоном 2xx (наприклад 404 чи 500). Нативний `fetch` не падає з помилкою на 404, він викидає помилку тільки при обриві мережі. Треба писати `if (!res.ok) throw new Error(...)`.
   - **Інтерсептори (Interceptors):** зручний спосіб додавати токени авторизації перед "кожним" запитом централізовано.

5. **Опишіть потенційні ризики багу "Стан гонитви" (Race Condition), якщо користувач буде дуже швидко перемикати сторінки з різними URL-параметрами, за якими робляться запити.**
   "Стан гонитви" виникає, коли кілька асинхронних запитів виконуються одночасно і невідомо, який з них поверне відповідь першим. Якщо користувач клацає "Категорія 1", а потім швидко "Категорія 2", браузер надсилає обидва запити. Якщо сервер обробляє "Категорію 2" швидше, вона відрендериться першою. Але потім "наздоганяє" застаріла відповідь "Категорія 1", і додаток відобразить дані "Категорії 1", хоча користувач дивиться на активну владку "Категорії 2". `AbortController` у `useEffect` елегантно вирішує цю проблему, відміняючи попередній запит перед запуском наступного.
