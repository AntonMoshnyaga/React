# Лабораторна робота №2

## 3. Фрагмент коду з App.jsx з використанням `.map()`:
```jsx
{postsData.map((post) => (
  <Post
    key={post.id} // КРИТИЧНО ВАЖЛИВО!
    author={post.author}
    content={post.content}
    date={post.date}
    avatar={post.avatar}
  />
))}
```

## 4. Що станеться, якщо видалити атрибут `key`:
Якщо видалити атрибут `key`, React виведе попередження (Warning) у консоль розробника браузера: `Warning: Each child in a list should have a unique "key" prop.`. Без унікальних ключів React не зможе ефективно відстежувати зміни в елементах списку (наприклад: додавання, видалення, зміна порядку). При змінах React буде перемальовувати список повністю або може неправильно зберегти локальний стан (state) елементів під час роботи алгоритму узгодження (Reconciliation).

## 5. Відповіді на контрольні запитання:

1. **Навіщо потрібен проп `key` у списках React?**
Проп `key` допомагає React ідентифікувати, які саме елементи масиву змінилися, були додані або видалені. Це надає елементам стабільну ідентичність і є критично важливим для оптимізації алгоритму Reconciliation, щоб оновлювати в DOM лише необхідні вузли.

2. **Чому не рекомендується використовувати індекс масиву (index) як ключ?**
Використання індексів масиву як ключів може призвести до проблем, якщо порядок елементів у масиві змінюється (наприклад, при фільтрації, сортуванні або додаванні нових елементів на початок списку). Це може викликати помилки рендерингу компонентів або втрату їх внутрішнього стану, оскільки компонент, що відносився до певного елемента, отримає новий індекс і змішається з іншим елементом.

3. **У чому різниця між імперативним створенням списку (цикл `for`) та декларативним (`map`)?**
Імперативний підхід (цикл `for`) описує *кроки виконання* (як це зробити): створення масиву, ітерація, пуш (push) елементів до масиву. Декларативний підхід (`.map()`) фокусується на тому, *який результат* потрібен, дозволяючи елегантно перетворити (трансформувати) масив даних у масив JSX-елементів прямо всередині розмітки компонента.

4. **Як передати всі властивості об'єкта в компонент одним рядком?**
За допомогою оператора розширення (spread operator): `<Component {...item} />`. Таким чином кожна властивість об'єкта `item` (наприклад, `item.name`, `item.age`) автоматично передається як окремий проп у `Component`.
