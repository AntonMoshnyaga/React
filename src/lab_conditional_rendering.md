# Практична робота: Логіка умовного відображення

## Відповіді на контрольні запитання

1. **Чому використання `if/else` безпосередньо всередині JSX-розмітки призведе до помилки?**
   JSX компілюється у виклики функцій (React.createElement). Так само як не можна вставити блок if/else безпосередньо всередину виклику функції як аргумент в JavaScript, це заборонено й у JSX. У фігурних дужках `{}` в JSX можна писати лише JavaScript *вирази* (expressions), які повертають значення, а не *інструкції* (statements) на зразок `if`, `for`, `while` чи `switch`.

2. **Яка небезпека криється у використанні числа `0` у лівій частині оператора `&&` в JSX?**
   Якщо в лівій частині `&&` знаходиться `0` (наприклад, `count && <Component />`), вираз обчислиться як `0`, оскільки оператор `&&` повертає перше `falsy` значення. Як наслідок, React безпосередньо відрендерить цифру `0` на екрані, замість того, щоб не показувати нічого. Розв'язанням є порівняння: `count > 0 && <Component />`.

3. **Як реалізувати умовний рендеринг, якщо логіка вибору компонента дуже складна (наприклад, 5+ варіантів)?**
   - Винести логіку в окрему функцію-помічник або метод перед return. Ця функція може використовувати `switch` або кілька `if/else` для повернення правильного компонента.
   - Використати об'єкт-словник (Object Map), де ключами є стани, а значеннями — JSX елементи (наприклад, `{ 'list': <List />, 'stats': <Stats /> }[state]`).

4. **В яких випадках використання тернарного оператора є кращим за використання оператора `&&`?**
   Тернарний оператор (`умова ? A : B`) ідеально підходить, коли вам потрібно показати один із *двох* взаємовиключних станів. Наприклад, "Кнопка входу" проти "Кнопки виходу" або "Завантаження..." проти готового контенту. Оператор `&&` краще застосовувати, коли умова бінарна: показати щось або нічого (hide/show).

5. **Як React розуміє, що потрібно перерендерити частину інтерфейсу при зміні вкладки?**
   Зміна вкладки відбувається шляхом виклику функції `setState` (наприклад, `setActiveTab('stats')`). Виклик оновлення стану повідомляє React, що дані компонента змінилися. React виконує повторний рендеринг компонента, генерує новий Virtual DOM, порівнює його зі старим Virtual DOM (алгоритм Reconciliation) і точково оновлює в браузері лише ті вузли, які реально змінилися (прихований таб видаляється з DOM, а новий монтується).
